# Contributing to HIE Integration Framework

Thank you for your interest in contributing! This framework aims to help healthcare organizations build reliable, compliant integration platforms.

## Ways to Contribute

### 1. Report Issues
- **Bugs**: If you find incorrect information or broken examples
- **Suggestions**: Ideas for new patterns or improvements
- **Questions**: Clarifications on implementation details

### 2. Submit Code
- Additional database-specific SQL examples (MySQL, SQL Server)
- Cloud-native deployment patterns (Kubernetes, AWS, Azure, GCP)
- Performance benchmarking scripts
- Additional message transformation examples
- Test cases and validation scripts

### 3. Improve Documentation
- Fix typos or unclear explanations
- Add real-world deployment experiences
- Translate to other languages

## Contribution Guidelines

### Code Style
- **Java**: Follow standard Java conventions, include Javadoc
- **SQL**: Use uppercase for keywords, lowercase for identifiers
- **Configuration**: Include detailed comments explaining each setting

### Documentation
- Include author attribution
- Explain the "why" not just the "how"
- Provide practical examples

### Commit Messages
```
[type] Brief description

Longer explanation if needed.

Fixes #123
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run any applicable tests
5. Commit with a descriptive message
6. Push to your fork
7. Open a Pull Request

### PR Checklist
- [ ] Code follows project style guidelines
- [ ] Comments/documentation updated
- [ ] No sensitive data (passwords, keys, PHI)
- [ ] License header included in new files
